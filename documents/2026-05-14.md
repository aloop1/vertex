# 회의록 (2026-05-14)

## 1. 회의 개요
- 프로젝트명: 소재부품 수명 예측 모델 개발
- 회의 목적: 제약 조건 내재화 기반 신합금 탐색 고도화, LMP 기반 데이터 증강 및 전처리 정합성 확보, 시뮬레이션 UI 연계 현황 점검

## 2. 금주 진행 사항

### 2.1 제약 조건 내재화 및 신합금 탐색 최적화
- 델타 페라이트 및 유해상 형성 경향을 Penalty로 반영한 종합 적합도 함수를 설계함.
- LLM 기반 초기 Seed 추출과 유전 알고리즘(GA)을 결합한 다목적 탐색 파이프라인을 구축함.
- 물리적 제약을 내재화해 단순 성능 중심 탐색이 아닌 제조 가능성과 안정성을 고려한 후보 탐색 체계를 확보함.

### 2.2 물리 법칙(LMP) 기반 데이터 증강 및 누수 방지 조건부 증강
- 동일 그룹 내에서 LMP를 활용해 기존 분포 범위 내 가상 샘플을 생성함.
- 데이터 누수 방지를 위해 신뢰도 조건(RMSE 0.35 이하)을 만족하는 Train Set에만 증강 로직을 적용함.
- 증강 데이터가 모델 학습에 기여하면서도 평가 신뢰도를 훼손하지 않도록 조건부 증강 원칙을 운영함.

### 2.3 전처리 파이프라인 정비 및 모델 정합성 전처리 구축
- 데이터 로드부터 추론 지원까지의 전처리 프로세스를 정비하고, 원소 표기 오류(Rh -> Re)를 수정함.
- 수명 예측 모델과 LMP 법칙 간 물리적 타당성 상관계수(R^2) 0.984를 달성함.
- 데이터 정합성 및 표기 일관성 개선을 통해 후속 학습/배포 단계의 재현성을 강화함.

### 2.4 시뮬레이션 및 시각화 UI 초안 마련
- 가상 격자 생성기를 기반으로 히트맵 렌더링 초안을 마련함.
- 모델 추론부와 Streamlit UI 연동 초안을 제작해 결과 확인 흐름을 구성함.
- 분석 결과의 직관적 전달을 위한 시각화 인터페이스 기반을 확보함.

## 3. 주요 의사결정 사항
- 탐색 엔진은 LLM Seed + GA 결합형 파이프라인을 기본 구조로 유지함.
- 데이터 증강은 조건부(신뢰도 기반) 정책으로 제한 적용해 누수 리스크를 관리함.
- 전처리 단계에서 원소 표기 및 스키마 정합성 검증을 필수 게이트로 운영함.
- 차주에는 증강 데이터 성능 검증, 앙상블 적용 타당성, UI 성능 최적화를 우선 수행함.

## 4. 리스크 및 대응
- 리스크 1: 증강 데이터가 특정 조건에서 편향을 확대할 수 있음
  - 대응: 원본 모델 대비 증강 모델 성능 비교와 조건별 에러 분석으로 적용 범위를 조정함.
- 리스크 2: 물리 제약 Penalty 설정이 과도하면 탐색 다양성이 저하될 수 있음
  - 대응: 페널티 가중치 자동 탐색과 민감도 분석을 병행해 균형점을 도출함.
- 리스크 3: UI 렌더링 지연으로 사용자 경험이 저하될 수 있음
  - 대응: Streamlit 렌더링 최적화, 캐싱 전략, 시각화 데이터 경량화를 적용함.

## 5. 차주 계획
- 원본 데이터 학습 모델과 LMP 증강 데이터 학습 모델 간 성능을 비교 분석함.
- 에러율이 높은 특정 조건에 대해 앙상블 기법 및 추가 물리 피처 엔지니어링 적용을 검토함.
- Streamlit 히트맵 렌더링 속도를 최적화하고 사용자 편의 기능을 확장함.
- 그룹별 LMP 피팅 그래프를 시각화하고, 합성 데이터 및 모델 결과 파일의 로컬 디렉토리 I/O 관리 체계를 안정화함.

## 6. 기대 산출물
- 조건부 LMP 증강 정책 문서 및 성능 비교 리포트
- 오차 조건 대상 앙상블/물리 피처 적용 검토 결과
- Streamlit UI/UX 개선 버전 및 렌더링 최적화 결과
- 그룹별 LMP 피팅 시각화 산출물 및 데이터 I/O 운영 가이드

## 7. 결론
- 금주에는 물리 제약을 내재화한 탐색 파이프라인과 LMP 기반 조건부 증강 체계를 구축하고 전처리 정합성과 UI 초안을 함께 확보해 모델 운영 기반을 강화함.
- 차주 핵심은 증강 데이터 실효성 검증과 UI 성능 고도화를 통해 실사용 가능한 분석 및 탐색 워크플로우로 전환.
