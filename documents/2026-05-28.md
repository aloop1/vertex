# 회의록 (2026-05-28)

## 1. 회의 개요
- 프로젝트명: 소재부품 수명 예측 모델 개발
- 회의 목적: GA 기반 신합금 설계 결과 정리 및 웹 대시보드 연동 계획 점검

## 2. 금주 진행 사항

### 2.1 GA 기반 신합금 설계 결과 도출 및 결과 파일 생성
- 수명 예측 모델과 연동되는 GA 기반 신합금 설계 파이프라인을 정리하고 대표 조건에서 최종 후보 합금 결과를 도출함.
- 최종 1등 후보 및 상위 후보군을 웹 대시보드에서 활용할 수 있도록 JSON 및 CSV 형태의 결과 파일로 정리함.

### 2.2 LLM seed cache 기반 초기 후보 생성 구조 안정화
- Gemini API를 매번 호출하지 않고 사전에 생성한 seed cache를 활용해 초기 후보 조성을 불러오도록 설정함.
- 이를 통해 API 호출 실패, 응답 지연, JSON 파싱 오류 없이 안정적으로 GA 최적화가 수행되도록 함.

### 2.3 CALPHAD 기반 phase validation 결과 정리
- pycalphad 기반 CALPHAD 검증을 적용하여 최종 후보의 상분율을 산출함.
- BCC_A2, M23C6, Laves phase 등 주요 상의 분율 정보를 결과 파일에 포함시켜 열역학적 안정성까지 함께 검증함.

### 2.4 웹 연동 방식 조정 및 대시보드 항목 정리
- GA 최적화와 CALPHAD 검증은 계산 시간이 길어 웹에서는 사전 생성된 결과 파일을 불러와 표시하도록 구성함.
- 대시보드는 1위 합금, 상위 후보군, 조성, 비용, CALPHAD 상분율 및 수명 예측값을 중심으로 시각화하도록 항목을 정리함.

## 3. 주요 의사결정 사항
- 결과 파일( JSON/CSV ) 기반 웹 연동 방식을 확정하여 대시보드 안정성을 우선 확보하기로 결정함.
- 초기 후보 생성은 seed cache 기반으로 운영하여 외부 API 의존도를 낮추기로 함.
- pycalphad 검증 결과를 필수 검증 항목으로 포함하여 후보의 물리적 타당성을 보강하기로 합의함.

## 4. 리스크 및 대응
- 리스크 1: GA 및 CALPHAD 계산의 높은 계산 비용
  - 대응: Top 후보군(예: Top 10) 중심의 후처리로 경량화하고, 필요 시 배치 사전 생성 전략을 적용함.
- 리스크 2: seed cache의 갱신·관리 문제
  - 대응: seed 버전 관리 정책과 재생성 주기 및 검증 파이프라인 수립
- 리스크 3: 웹에서 동적 검증 불가로 인한 최신성 문제
  - 대응: 결과 파일 생성 시 타임스탬프 및 데이터 버전 표기, 사용자에게 최신 결과 재요청 기능 제공

## 5. 차주 계획

### 5.1 웹 대시보드 연동 및 결과 시각화 보완
- 결과 파일을 기반으로 신합금 설계 결과 대시보드를 구성하고, 상위 후보군의 상세 뷰(조성, 상분율, 예측수명)를 추가함.

### 5.2 GA 실행 시간 개선 및 후보 검증 고도화
- GA 탐색의 계산 비용을 낮추기 위해 Top 10 후보 검증 중심으로 경량화하고, 병렬 처리 옵션을 검토함.

### 5.3 최종 발표 자료 구성 및 시스템 통합 점검
- 수명 예측, 후보 생성, GA 탐색, CALPHAD 검증의 전체 흐름을 포함하는 최종 발표 자료 초안을 작성하고, 웹 대시보드에서 결과가 올바르게 표시되는지 통합 점검을 수행함.

## 6. 기대 산출물
- GA 기반 신합금 설계 결과 JSON/CSV 파일(Top 후보군 포함)
- pycalphad 기반 상평형 검증 리포트 및 상분율 표
- 웹 대시보드용 시각화(상위 후보 상세 뷰 포함)
- 최종 발표 자료 초안

## 7. 결론
- 금주에는 GA 파이프라인의 결과물 생산과 CALPHAD 검증을 통해 신합금 후보의 물리적 타당성 검토 프로세스를 정비함.
- 다음 주에는 대시보드 연동과 실행 시간 개선에 중점을 두어 사용자 검증 편의성과 시스템 응답성을 향상시킬 예정임.
